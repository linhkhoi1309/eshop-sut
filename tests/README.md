## Run
```bash
# 1) Backend up on :3000 with a fresh DB (server.js reseeds on boot):
cd backend && npm install && node server.js   # keep running

# 2) From the repo root, run the suite (Node 18+):
node tests/runner.mjs

# options
ESHOP_API=http://192.168.1.10:3000 node tests/runner.mjs   # non-default backend
ESHOP_FEATURE=FR-09 node tests/runner.mjs                  # one feature only
ESHOP_ALLOW_DB_RESET=1 node tests/runner.mjs               # allow resetDb() (destructive)
```
Then hand `tests/results.json` to the **run-tests-and-report** skill to produce
`docs/test-results.md` (summary + candidate-bug table).

## Determinism notes (important)
- **Start from a freshly reseeded DB** (`node backend/database.js`, or just restart `server.js`).
  Coupon-usage boundaries use **throwaway coupons** (`makeTempCoupon`) so they don't depend on
  shared state, but categories/orders cases create rows — a clean start keeps counts predictable.
- **Fixtures build their own state**: order-state cases checkout + drive via the admin API;
  the FR-11 case registers a second user; expiry cases use runtime-relative dates (`dateOffset`).
- A few report steps were adapted for non-destructive automation (noted in `cases.mjs`), e.g.
  `FR05-BVA-07` uses a one-char-off term instead of deleting the Keychron product; the **Expected
  (spec)** is preserved.
